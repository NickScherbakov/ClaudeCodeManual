*Черновик статьи для Habr. Публиковать после 1000 звезд или по достижению промежуточной цели.*

---

Когда задача слишком большая для одного промпта — большинство людей просто разбивают её на несколько вопросов и работают вручную между ними. Это нормально для маленьких задач. Но что делать, когда нужно:

- Проанализировать репозиторий из 300 файлов на уязвимости
- Проверить каждую найденную уязвимость независимым агентом-скептиком
- Продолжить с точки остановки, если что-то упало на середине
- Масштабировать глубину анализа в зависимости от доступного бюджета

Это уже не задача для одного промпта. Это задача для **Workflow tool** — встроенного оркестратора многоагентных систем в Claude Code.

## Что такое Workflow

Workflow — это JavaScript-скрипт, который описывает структуру работы нескольких агентов. Вы передаёте его Claude Code, и он выполняет его детерминированно: с прогресс-баром, с возможностью возобновить с точки остановки, с параллельным запуском агентов.

```javascript
export const meta = {
  name: 'code-audit',
  description: 'Аудит кодовой базы на уязвимости',
  phases: [
    { title: 'Поиск' },
    { title: 'Верификация' },
  ]
}

phase('Поиск')
const findings = await pipeline(
  files,
  file => agent(`Найди уязвимости в ${file}`, { schema: VULN_SCHEMA })
)

phase('Верификация')
const verified = await parallel(
  findings.flat().filter(Boolean).map(f => () =>
    agent(`Попробуй опровергнуть: ${f.title}. По умолчанию считай, что это ложная тревога.`, {
      schema: VERDICT_SCHEMA
    })
  )
)

return verified.filter(v => !v.refuted)
```

Запускается одной командой:
```
Запусти project/code-audit.js как workflow +100k
```

## Ключевые концепции, которые меняют подход

### 1. Проектируйте от цели, не от инструмента

Стандартная ошибка: "у меня есть 200 файлов, что с ними можно сделать?"

Правильный вопрос: "что должно существовать, когда работа закончена?"

Это разница между:
- "агент обрабатывает файлы" (описание процесса)
- "существует отчёт с подтверждёнными уязвимостями, отсортированными по критичности" (описание результата)

Когда результат ясен, декомпозиция на задачи становится очевидной.

### 2. pipeline() — по умолчанию, parallel() — только при необходимости

`pipeline()` пропускает каждый элемент через все стадии по очереди. Пока элемент A на стадии верификации, элемент B уже начинает анализ. Нет барьеров между стадиями.

`parallel()` — барьер: ждёт все результаты стадии N перед началом N+1. Нужен только когда стадия N+1 действительно требует все результаты N сразу (дедупликация, ранний выход, сравнение).

```javascript
// ✅ pipeline — когда каждый элемент проходит путь независимо
const results = await pipeline(
  documents,
  doc => agent(`Проанализируй ${doc}`, { schema: ANALYSIS }),
  analysis => agent(`Верифицируй: ${analysis.summary}`, { schema: VERDICT })
)

// Используйте parallel() как барьер только когда нужны ВСЕ результаты сразу
const allFindings = await parallel(SOURCES.map(s => () =>
  agent(s.prompt, { schema: FINDINGS })
))
const deduped = dedupe(allFindings.filter(Boolean).flat())
```

### 3. seen set — не accumulated results

В динамических циклах (loop-until-dry) дедупликация должна идти через `seen` — набор всех виденных кандидатов, включая отклонённых верификацией:

```javascript
const seen = new Set()
let dry = 0

while (dry < 2) {
  const result = await agent('Найди новые риски', { schema: RISKS })
  const fresh = result.items.filter(r => !seen.has(r.id))

  if (!fresh.length) { dry++; continue }
  
  dry = 0
  fresh.forEach(r => seen.add(r.id))  // запоминаем кандидата, даже если верификация его отклонит
  risks.push(...fresh)
}
```

Если дедуплицировать только по принятым результатам, отклонённые находки будут возвращаться в каждом раунде вечно.

### 4. Бюджетный guard

```javascript
// ❌ Без бюджета: remaining() = Infinity → бесконечный цикл
while (budget.remaining() > 50_000) { ... }

// ✅ Правильно
while (budget.total && budget.remaining() > 50_000) { ... }
```

## Комментарий к статье

Что оказалось неочевидным, когда я начинал с Workflow tool?

- Скрипты — это JavaScript, но **без `node`**. `Date.now()` и `Math.random()` недоступны, потому что ломают механизм возобновления.
- `parallel()` возвращает `null` для упавших агентов, а не выбрасывает исключение. Всегда `.filter(Boolean)`.
- `meta.phases[].title` должны совпадать с вызовами `phase('...')` дословно.
