# Lab 09 — Система памяти

**Сложность**: ★★☆
**Время**: ~60 минут

---

## Цель

Разобраться с системой персистентной памяти Claude Code: 4 типа памяти, когда каждый нужен, и жизненный цикл — когда сохранять, обновлять и удалять.

---

## Концепция: Зачем память?

Без памяти каждый разговор с Claude начинается с нуля. Claude не знает, кто ты, что ты предпочитаешь, каков контекст проекта и где искать информацию.

С памятью Claude помнит между сессиями — не нужно каждый раз объяснять одно и то же.

---

## Четыре типа памяти

### 1. `user` — профиль пользователя

**Что сохранять**: роль, опыт, ответственности, знания  
**Цель**: адаптировать ответы к уровню пользователя

```markdown
---
name: user-profile
description: Разработчик Python с опытом ML, новый в Claude Code
metadata:
  type: user
---

Senior Python разработчик (10 лет). Специализация: ML/data science.
Новый в Claude Code и workflow-скриптах.
Prefer: объяснения с аналогиями из Python, не с JavaScript.
```

### 2. `feedback` — правила взаимодействия

**Что сохранять**: коррекции ("не делай X"), подтверждения ("да, именно так")  
**Структура**: Правило + **Why:** причина + **How to apply:** применение

```markdown
---
name: feedback-response-style
description: Не добавлять summary после каждого ответа
metadata:
  type: feedback
---

Не резюмировать действия в конце каждого ответа.
**Why:** Пользователь может читать diff сам, summary создаёт лишний шум.
**How to apply:** Заканчивать ответ действием, не объяснением того что было сделано.
```

### 3. `project` — контекст проекта

**Что сохранять**: цели, решения, ограничения, дедлайны  
**Структура**: Факт + **Why:** мотивация + **How to apply:** как влияет

```markdown
---
name: project-auth-rewrite
description: Переписываем auth middleware из-за compliance требований
metadata:
  type: project
---

Auth middleware переписывается. Дедлайн: 2026-07-01.
**Why:** Legal потребовал убрать хранение session tokens в Redis — не соответствует новым compliance требованиям.
**How to apply:** Все новые решения должны использовать stateless JWT. Не предлагать Redis-based решения для сессий.
```

### 4. `reference` — указатели на внешние ресурсы

**Что сохранять**: где хранятся данные в внешних системах

```markdown
---
name: reference-jira-board
description: Задачи команды в Jira проекте PLATFORM
metadata:
  type: reference
---

Задачи бэкенд-команды: Jira проект PLATFORM.
Баги: фильтр по label "backend-bug".
Спринт доски: PLATFORM/boards/3.
```

---

## MEMORY.md — индекс памяти

```
~/.claude/projects/[project-path]/memory/MEMORY.md
```

Каждая запись — одна строка ≤150 символов:

```markdown
- `user-profile.md` — Senior Python dev, ML focus, new to Claude Code
- `feedback-response-style.md` — no trailing summaries, terse responses
- `project-auth-rewrite.md` — auth rewrite due to compliance, deadline 2026-07-01
- `reference-jira-board.md` — backend tasks at Jira/PLATFORM
```

**ВАЖНО**: MEMORY.md всегда загружается в контекст. После 200 строк — обрезается. Держи кратким.

---

## Что НЕ сохранять в память

Память НЕ для:
- Кодовых паттернов (читай код напрямую)
- Git истории (используй `git log/blame`)
- Отладочных решений (они в коде/коммитах)
- Уже задокументированного в CLAUDE.md
- Временных деталей текущей задачи

**Тест**: "Это будет полезно через месяц, если забыть весь текущий контекст?" — если нет, не сохраняй.

---

## Жизненный цикл памяти

### Когда сохранять:
- Коррекция поведения: "не делай X" → feedback немедленно
- Профиль пользователя изменился: новая роль, новые знания → обновить user
- Важное решение по проекту → project
- Найден новый внешний ресурс → reference

### Когда обновлять:
- Fact в project memory устарел
- Feedback изменился (пользователь изменил предпочтение)
- Дедлайн прошёл → удалить или обновить

### Когда удалять:
- Проект завершён
- Правило отменено
- Пользователь попросил забыть

### Перед использованием:
**Верифицируй** — памяти устаревают. Если память упоминает файл → проверь, что он существует. Если запись противоречит реальному состоянию → доверяй реальному и обнови память.

## Задания

### Задание 9.1: Создай user memory
Попроси Claude запомнить информацию о тебе:
```
"Запомни: я [описание роли], специализируюсь на [область], 
предпочитаю [стиль ответов]"
```
Проверь что Claude создал файл в memory директории.

### Задание 9.2: Дай feedback
Попроси Claude сделать что-то "неправильно" (например, написать длинный summary), 
а потом скажи что это не нужно. Убедись что Claude создал feedback memory.

### Задание 9.3: Проверь persistence
Начни новый разговор (или `/clear`) и проверь что Claude помнит твои предпочтения.

### Задание 9.4: Обнови устаревшую память
Если у тебя есть memory с датой — измени дату в прошлое и спроси Claude про неё. 
Проверь обновит ли он запись.

```bash
# Посмотреть все файлы памяти:
/memory

# Директория памяти (Windows):
%USERPROFILE%\.claude\projects\[проект]\memory\

# Директория памяти (Mac/Linux):
~/.claude/projects/[проект]/memory/
```
