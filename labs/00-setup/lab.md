## Цель

Проверить окружение, разобраться в структуре тренажера и понять как строится обучение.

## Задание 1: Проверь окружение

Выполни эти команды в Claude Code и убедись, что всё работает:

```bash
# 1. Версия Claude Code
claude --version

# 2. Текущая модель
/config

# 3. Доступные команды
/help
```

**Критерий успеха**: Видишь версию Claude Code, можешь открыть настройки.

---

## Задание 2: Настрой Opus 4.8

Claude Opus 4.8 — самая мощная модель для workflow. Установи её:

```bash
# В интерфейсе Claude Code:
/config
# Найди поле model, установи: claude-opus-4-8
```

Или через settings.json:
```json
{
  "model": "claude-opus-4-8"
}
```

**Где лежит settings.json**:
- Windows: `%APPDATA%\Claude\settings.json`
- Mac/Linux: `~/.claude/settings.json`
- Проект: `.claude/settings.json` (перекрывает глобальные)

**Критерий успеха**: В строке статуса отображается `opus-4.8` или `claude-opus-4-8`.

## Концепция: Как устроен Claude Code

Claude Code — это CLI и IDE-расширение для работы с Claude. Ключевые слои:

```
Пользователь
    ↓
Claude Code CLI / IDE Extension
    ↓
Claude API (модель: claude-opus-4-8)
    ↓
Tools: Read, Write, Edit, Bash, Glob, Grep, Agent, Workflow...
    ↓
Filesystem / External Services / Subagents
```

**Уровни автономности** (настраиваются в settings.json):
- `plan` — Claude предлагает план, ты подтверждаешь каждое действие
- `auto` — Claude действует автономно в рамках разрешений
- `manual` — ты подтверждаешь каждый инструмент
