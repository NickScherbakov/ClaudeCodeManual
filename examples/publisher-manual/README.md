# Сквозной пример: руководство для издательства

Этот пример показывает весь курс на одной практической задаче.

Главный герой получает заказ от издательства: подготовить практическое руководство
по продвинутым техникам Claude Code для авторов учебников. Издательство просит
вести работу в репозитории своего GitHub-профиля, чтобы редактор, методист и
технический рецензент видели ход проекта.

## Сюжет

1. Издательство присылает первый оффер и требования к книге.
2. Автор создаёт репозиторий `edu-press/claude-code-textbook-guide`.
3. В репозиторий попадают бриф, письма редактора, требования методиста,
   замечания технического рецензента и черновой план.
4. Claude Code помогает превратить эти материалы в концепцию книги.
5. Рабочий процесс извлекает требования, строит структуру, проверяет пробелы,
   собирает план глав и формирует пакет сдачи.

## Вход

`source-materials.md` — искусственный набор материалов проекта: оффер
издательства, письма редактора, требования к методике, технические ограничения
и заметки автора.

## Запуск

В Claude Code:

```text
Запусти examples/publisher-manual/publisher-manual-workflow.js как workflow +150k
```

## Что должен сделать рабочий процесс

1. Прочитать `source-materials.md`.
2. Вытащить требования издательства, ограничения, риски и критерии готовности.
3. Сформулировать идеальный результат: какая книга должна существовать в конце.
4. Разложить работу на главы, проверки и артефакты репозитория.
5. Проверить, чего не хватает для практического руководства.
6. Собрать пакет сдачи: концепция, оглавление, план глав, контрольный список и
   следующие действия в GitHub-репозитории.

## Почему это сквозной пример

- `agent()` извлекает требования из писем и заметок.
- `schema` превращает редакционные требования в проверяемые данные.
- `parallel()` даёт взгляды редактора, методиста и технического рецензента.
- Проверка полноты ищет отсутствующие разделы книги.
- Финальный синтез превращает разрозненные материалы в план готовой рукописи.
