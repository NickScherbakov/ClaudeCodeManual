# Примеры реальных Workflow-систем

Полноценные workflow-скрипты для учебных и рабочих задач.

> Это **скрипты для Workflow tool** — запускать через Claude Code, не через `node`.

---

## `system.js` — Аудит кодовой базы

Демонстрирует полный цикл: многоизмерительный анализ → дедупликация → адверсариальная верификация → отчёт.

Анализирует `target/app.js` — намеренно проблемный файл с SQL injection, XSS, небезопасными токенами и другими уязвимостями. Это учебная мишень, не рабочий код.

```
Запусти project/system.js как workflow +100k
```

## `infolimp-audit.js` — Аудит информационных потоков

Workflow для анализа структуры информационного сайта: находит пробелы в охвате тем, дублирование, слабые места.

```
Запусти project/infolimp-audit.js как workflow +80k
```

## `trainer-improve.js` — Улучшение тренажёра

Самоаудит: workflow анализирует главы и лаборатории тренажёра и предлагает улучшения.

```
Запусти project/trainer-improve.js как workflow +80k
```

---

## `target/app.js` — учебная мишень

Намеренно проблемный файл Node.js. Содержит:
- SQL injection в аутентификации
- XSS в рендеринге
- Небезопасная генерация токенов
- Другие учебные уязвимости

Используется как объект анализа в `system.js` и лаб 06, 07, 13. **Не исправляйте вручную** — ошибки intentional.
